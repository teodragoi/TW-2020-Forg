Introduction

GarT is a web application meant to help the everyday gardner that wants to get started to learn a few useful things about the trade. Using this app, you can select any course from the wide variety that is offered, in order to pick just the right one for what you want to learn.

Of course, what fun is learning if there isn't a bit of competition involved. So in order to insert this new competitive element we also present a ranking of all our users, based on the number of courses they have completed. So pick up your tools and get on started with the learning. Make sure you get among the first top ten students of our online platform.

Structure

In order to not mix up our functionalities we have separated the folder structure in two: front-end and back-end.

The front-end presents a friendly GUI, build on technologies such as: HTML5, JavaScript and CSS3 meant to present all of our functionalities as clearly as possible.

On the other hand, the back-end is where all the "nasty" stuff is happening. We created a NodeJS server, that uses a Mongo Database to store all data necessary for the smooth running of our application.

Front-end

To make our code as clean as possible, we separated the files used on the front-end side of things into five:

  • assets
  • models
  • pages
  • scripts
  • styles

Each file in each folder serves to a specific type of functionality:

  • assets - This is where we keep all photos used in the project
  • models - This is where all classes that are used when mapping our resources from the backend find themselves
  • pages - This folder contains the HTML code for our statically generated elements in each page
  • scripts - This is where JavaScript files that handle either requests or dynamic generation of HTML elements are
  • styles - And finally, in styles is where all the magic that makes each page look awesome(at least by our tastes) is happening

Back-end

The back-end is much simpler to put into words, as it has only two folders and 1 file that are absolutely necessary:

  • db
  • models
  • index.js

As is with the front-end, each folder in the back-end is meant for one thing, and one thing only:

  • db - This is where the db.js file is found, which is responsible for every interaction that the server needs to have with the database
  • models - This folder contains all models that we use in order to map either responses from the database or request bodies
  • index.js - Here is the birthplace of our server, which handles requests sent by the client

Endpoints

Our back-end side of things exposes a series of endpoints, meant for being of service to the Client:

GET

  • /Courses - returns all existig Courses
  • /Courses?courseId=id - returns the course that has the _id === id
  • /Courses?searchString="searchString" - returns the course for which either the author or the title bears some resemblance to the searchString
  • /Users?username="username" - returns the user which has the Username "username"
  • /Users?userRanking="username" - returns the ranking of the user with the Username "username"
  • /Users/Rankings - returns the top 10 users of our platform

POST

  • /Courses - creates a new Course based on the request body
  • /Users - creates a new User based on the request body
  • /Users/login - logs a user in based on the request body
  • /Users/Update - updates the count of how many courses the user has completed

Request bodies

  • Course: { title: "Course title", author: "Course author", materials: ['mat1', 'mat2', 'mat3'], tasks: ['task1', 'task2', 'task3'] }
  • Users: { username: "Some Username", password: "Some Password", admin: false, coursesCompleted: 0 }
  • Users login: { username: "Some Username", password: "Some Password" }
  • Users update course count: { username: "Some Username", coursesCompleted: 3 }
